Introduction

Base URL: https://api.raffled.live 
This documentation aims to provide all the information you need to work with our API.

As you scroll, you will see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile).
You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include a Authorization header with the value "Bearer {YOUR_AUTH_TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can obtain your token by completing the SIWE process using the /siwe/nonce and /siwe/verify endpoints.

Fee Change Events

List on-chain fee change events.

Example request:
curl --request GET \
    --get "https://api.raffled.live/api/fee-change-events?tx_hash=consequatur&per_page=20&sort_dir=consequatur" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "https://api.raffled.live/api/fee-change-events"
);

const params = {
    "tx_hash": "consequatur",
    "per_page": "20",
    "sort_dir": "consequatur",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

}).then(response => response.json());

requires authentication

curl --request POST \
    "https://api.raffled.live/api/raffles" \
    --header "Authorization: Bearer {YOUR_AUTH_TOKEN}" \
    --header "Content-Type: multipart/form-data" \
    --header "Accept: application/json" \
    --form "raffle_tx_hash=0xabc123"\
    --form "description=Dolores dolorum amet iste laborum eius est dolor."\
    --form "image=@C:\laragon\tmp\php17DF.tmp" 
const url = new URL(
    "https://api.raffled.live/api/raffles"
);

    "Authorization": "Bearer {YOUR_AUTH_TOKEN}",
    "Content-Type": "multipart/form-data",
    "Accept": "application/json",
};

Request      

POST api/auth/verify

Headers

Content-Type        

Example: application/json

Accept        

Example: application/json

Body Parameters

message   string     

Message containing Address and Nonce, signed via personal_sign (Base) or ed25519 (Solana). Example: consequatur

signature   string     

Message signature. Base58 for Solana, 0x-hex 65-byte for Base. Example: consequatur

address   string     

Signer public key. Base58 for Solana, 0x-hex for Base. Example: consequatur

chain   string     

Network. One of: solana, base. Example: consequatur
